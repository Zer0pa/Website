# GGD PROJECT

## Program

`GGD` means `Get Geometry Done`.

This is the repo-local adaptation of `get-physics-done` for deterministic website engineering.

The core question is:

How do we turn the Zer0pa website into a deterministic geometry program whose layout, typography, color, semantic hierarchy, data truth, and code quality are governed by explicit laws and falsified continuously until canonical closure is reached?

## Scope

In scope:

- homepage
- flagship `/imc`
- generic `/work/[slug]` family
- route truth from GitHub-derived packets
- semantic hierarchy, metadata, SEO, and code quality
- responsive behavior across declared viewport matrix
- augmentation only after canonical closure

Out of scope:

- production deployment
- remote control surfaces
- work outside `/Users/zer0palab/Zer0pa Website/Website-main`
- speculative redesign not justified by declared geometry law

## Decisive Outputs

The program is only succeeding if it produces all of these:

1. Canonical routes with zero critical and zero major geometry failures.
2. A convention lock that prevents creative drift.
3. A verification system that catches geometry, truth, semantic, and code-quality regressions.
4. A route-family kernel that generalizes across repo-backed product pages.
5. An autonomy layer that can recurse without degrading quality.

## Route Roles

- `home`: landing route and narrative shell
- `flagship`: `/imc`, with explicit non-generic exceptions
- `product-family`: generic repo-backed `/work/[slug]` routes

## Authoritative Inputs

- mockup references in the website repo
- route specs in `site/src/lib/layout/specs.ts`
- measurement and diff outputs in `deterministic-design-system/`
- packet truth in `site/src/lib/data/` and runtime cache
- current autonomy evidence in `CLAW/control-plane/`

## Current Known Truth

- homepage and `/imc` are the strongest closed routes
- the current control plane can run bounded cycles and settle correctly
- `/work/xr` still fails real geometry falsification
- the current system is not honest to leave unattended yet

## Program Rule

The website is not a mood board.

It is a deterministic system:

`route_output = f(constants, route_role, packet_truth, viewport, verification_contract)`
