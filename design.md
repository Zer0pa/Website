### **ZER0PA — DESIGN.md**

**Version:** 1.1  
**Date:** 27 March 2026  
**Scope:** zer0pa.ai public website

**Primary use:** Stitch initialisation \+ downstream Next.js \+ Sanity implementation

#### **1\. Core Directive**

Design **zer0pa.ai** as the canonical **proof-first authority surface** for Zero-Point Architecture.

This is **not** a startup landing page, repo gallery, or philosophy-first manifesto.

It is a silent, generational technical substrate that feels like a **DARPA leak presented with elite editorial discipline**.

The emotional result within 10 seconds must be:

* “There is something real here.”  
* “This is a disciplined system, not a pile of experiments.”  
* “Every claim has boundaries and proof.”  
* “Everything emerges from the zero.”

#### **2\. Brand Substrate (Non-Negotiable)**

* **Wordmark:** Oswald Regular or Medium, all caps. ZER0PA with white zero as the singularity/portal (no outline, no holder).  
* **Typography:**  
  * Mastheads / wordmarks → Oswald only  
  * All UI, body, data, evidence, metadata, labels → Courier (or equivalent monospace)  
* **Colour Palette:**  
  * Background: pure black (\#000000) — the void, origin, infinite potential  
  * Body / neutral text: greyscale (neutral, understated intelligence)  
  * Emphasis / singularity: white (reserved exclusively for the zero and critical proof moments)  
* **Visual Character:** Swedish minimalism meets high-technology restraint. Atomic. Embryonic by design. Nothing extraneous. Every element must justify its existence and trace back to the zero.  
* **Forbidden:** gradients, cyberpunk noise, decorative particles, chrome, rainbow palettes, faux-hacker terminals, marketing whimsy.

#### **3\. Product Strategy**

* Evidence leads. Architecture second. Philosophy third (compressed).  
* Homepage hierarchy: one flagship (ZPE-IMC) → supporting proof lane (ZPE-IoT) → constellation of bounded lanes.  
* Every page must separate **what is proved** from **what is explicitly not claimed**.  
* Site must be living and updateable without redesign.

#### **4\. Site Map (Required Routes)**

* / — Homepage (authority surface)  
* /imc — Flagship deep-dive  
* /work — Constellation index  
* /work/\[slug\] — Reusable mini-authority page template  
* /proof — Evidence model & audit routes  
* /about — Compressed substrate narrative  
* /contact — Partnership/licensing/research routing

#### **5\. Interaction Design Rules**

* **Motion:** Only to explain structure, hierarchy, causality, or lane convergence. Zero as activation/aperture point. Scroll-linked storytelling only. Full reduced-motion support.  
* **3D:** React Three Fiber — sparse, on-demand, explanatory only (e.g. zero-point singularity in hero, system architecture model). No decorative use.  
* **Cursor/Hover:** Subtle, precise, technical. Never playful.  
* **Layout Grammar:** Strict grid, large negative space, modular evidence cards, architectural rhythm.

#### **6\. Homepage Modules (Exact Order for Stitch)**

1. Hero (wordmark \+ authority headline \+ restrained 3D explainer)  
2. Flagship authority block (ZPE-IMC)  
3. Work constellation grid  
4. System architecture explainer (scroll-linked)  
5. Compressed proof logic  
6. Substrate narrative (3–5 lines max)  
7. Operations footer

#### **7\. Work Detail Page Template (Reusable)**

1. Hero \+ masthead  
2. What this is  
3. Current authority state  
4. Headline metrics  
5. What is proved now  
6. What is explicitly **not** claimed  
7. Evidence & proof routes  
8. Repo shape / verification path  
9. Related lanes  
10. Partnership CTA

#### **8\. Evidence & Data Visualisation Rules**

* Every claim must attach to a proof packet, benchmark, or repo truth.  
* Visuals: D3 for bespoke diagrams \+ Observable Plot for clean benchmarks. Monochrome with texture/line variation. Every chart must have a text summary.  
* Status badges, boundary callouts, and claim/non-claim splits must be visually prominent.

#### **9\. Tone & Content Rules**

* Exact, sparse, technical, confident, non-hype.  
* Bounded claims only. Explicit non-claims required.  
* No “revolutionary”, “world-class”, “game-changing”.

#### **10\. Technical & Accessibility Rules**

* WCAG 2.1 AA compliance  
* Keyboard navigability, visible focus states, reduced-motion mode  
* Core Web Vitals targets (INP ≤ 200 ms, TTFB ≤ 0.8 s)  
* Static-first rendering with lazy-loaded 3D and dense modules

#### **11\. Stitch-Specific Instructions**

When generating in Stitch:

* Generate **3 distinct homepage directions** in parallel.  
* Use real copy, real metrics, real authority states and boundaries.  
* Attach this DESIGN.md \+ ZER0PA logo SVG/PNG \+ brand ethos screenshots \+ masthead examples as visual context.  
* Stay strictly within the monochrome black/grey/white palette.  
* Prioritise clarity, authority, and brand discipline above aesthetic flair.
